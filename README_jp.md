# PhantomCipher (ファントムサイファー)

[English](./README.md) | [简体中文](./README_zh.md) | [日本語](./README_jp.md)

---

PhantomCipher は、Obsidian 向けにシームレスな透明暗号化体験を提供します。

> [!CAUTION]
> **警告と免責事項**
> - **開発初期段階**：このプロジェクトは初期段階にあり、暗号化アーキテクチャが予告なく変更される可能性があります。
> - **機密データ非推奨**：このプラグインは高度な機密情報や基幹業務データを扱うようには設計されていません。Obsidian の Secret Storage は、一部のプラットフォームで物理的に完全に隔離されているわけではないことに注意してください。
> - **バックアップの必須**：重要なファイルを暗号化する前に、**必ずヴォルト全体のバックアップを取ってください**。いかなるデータの紛失についても責任を負いません。
> - **資格情報の保存**：KEK (鍵暗号化鍵) と DEK (データ暗号化鍵) を手動で保存してください。複数端末で同期する場合は、両方の端末で同じ KEK/DEK を設定する必要があります。
> - **パスワードを忘れると、データは永久に復元できません。**

### ✨ 主な機能

- **エンベロープ暗号化 (V2)**：KEK で DEK を保護する設計。物理ファイルを書き換えることなくパスワード変更が可能です。
- **透過的なワークフロー**：低レベルの I/O をインターセプトします。エディタでは通常通り読み書きでき、ディスク上では暗号化されて保存されます。
- **圧縮機能内蔵**：Deflate アルゴリズムを使用して、Base64 エンコードによるファイルサイズの増加を抑制します。
- **大容量ファイル対応**：最大 2GB までのファイルをサポート。JS の制限と安全な変換プロセスにより、大容量ファイルを扱う場合は 16GB 以上のメモリを搭載したデバイスを推奨します。
- **バイナリストレージ形式**：構造はバイナリ形式での保存に対応し、メディアファイルの読み込み効率を大幅に向上させました。
- **整合性チェック**：手動変換プロセスに **MurmurHash3 平文ハッシュ検証** を導入。暗号化前後のデータ整合性を保証し、破損を未然に防ぎます。


### 🚀 クイックスタート

1. 設定画面で **マスターパスワード管理** を選択します。
2. モーダルでパスワードを設定します。KEK が派生され、ランダムな DEK が生成されて Obsidian のシークレットストレージに保存されます。
3. **重要**：Obsidian 設定フォルダ内の `secrets.json` をバックアップするか、生成されたキー ID を記録してください。
4. **動作モード** を「自動暗号化」に設定します。
5. **リボンアイコン** または **右クリックメニュー** を使用して、ファイルやフォルダの状態を切り替えます。
