# PhantomCipher (幻影加密)

[English](./README.md) | [简体中文](./README_zh.md) | [日本語](./README_jp.md)

---

PhantomCipher 为您的 Obsidian 库提供无缝的透明加密体验。

> [!CAUTION]
> **风险提示与免责声明**
> - **早期阶段**：本项目仍处于早期开发阶段，加密架构可能随版本更新而发生重大变更，请密切关注更新日志。
> - **非高度安全**：本插件并非为处理高度机密或核心业务数据设计。请注意，Obsidian 提供的钥匙串接口在特定平台下的存储并不具备强物理隔离性。
> - **必须备份**：在处理任何重要文件前，**请务必对原始仓库进行完整备份**。作者不对任何形式的数据丢失或损坏负责。
> - **凭据备份**：请务必手动记录并保存您的 KEK (密钥加密密钥) 和 DEK (数据加密密钥) 字符串。若需多端同步，请确保各端的 KEK/DEK 保持一致，否则将导致文件被隔离锁定。
> - **遗忘密码将导致文件永久无法找回。**

### ✨ 核心特性

- **信封加密架构 (V2)**：采用 KEK 保护 DEK 的设计，支持快速更改密码而无需重写物理文件。
- **透明加解密**：拦截底层读写逻辑；你在编辑器中正常书写，文件在磁盘上以密文存储。
- **内置压缩**：使用 Deflate 算法，有效抵消 Base64 编码带来的体积膨胀。
- **大文件支持**：支持最高 2GB 的附件加解密预览。由于 JS 环境限制与安全转换机制，建议在内存大于 16GB 的设备上处理超大附件。
- **二进制原生格式**：结构支持直接以二进制格式存储加密载荷，大幅提升媒体文件的读取效率并减少体积膨胀。
- **安全转换校验**：手动转换流程引入 **MurmurHash3 明文散列校验**，确保加解密前后的数据完整性，严防数据损坏。



### 🚀 快速开始

1. 在插件设置中找到 **管理主密码**。
2. 在弹出的窗口中设置您的主密码。系统将自动派生 KEK 并生成随机 DEK 存入 Obsidian 密钥串。
3.  **重要（凭据导出）**：由于密钥存储在钥匙串中，**请务必手动记录并导出**钥匙串密钥列表界面显示的以 `phantom-kek-` 和 `phantom-dek-` 开头的密钥值**。建议将这些值存放在离线的密码管理器中。
4. 将 **运行模式** 设置为“自动加密”。
5. 通过 **功能区图标** 或 **右键菜单** 转换单个文件或文件夹的状态。
